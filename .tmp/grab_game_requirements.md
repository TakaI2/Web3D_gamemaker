# 振り回しキャッチ（仮称: Swing Catch）要件定義

## 1. 概要

FPS（一人称）視点で、空中を飛び回るオブジェクトを相手に遊ぶサンドボックスゲーム。
プレイヤーは以下の2つの操作でオブジェクトに干渉する。

- **右クリック**: カメラ前方に球を発射。命中したオブジェクトを運動量で吹っ飛ばす。
- **左クリック（押している間）**: 照準先のオブジェクトをキャッチ。スプリングで繋がったように
  視点（マウス）を振るとオブジェクトがブンブン追従する。勢いよく振り回した状態で離すと、
  その時の速度でオブジェクトが飛んでいく。

まずは「勝敗なしのサンドボックス」として、操作感（振り回し・吹っ飛ばし）の気持ちよさを最優先に作る。
スコアやルールは操作感が固まった後に追加検討する。

## 2. 技術前提

- Three.js v0.184（`esm.sh` CDN）／**WebGPURenderer + TSL**（既存デモと統一）
- 物理は**自前実装**（固定タイムステップ積分・球/壁の単純衝突・スプリング力・投擲の運動量保存）
- 物理エンジン（Rapier等）は導入しない
- 配置: 既存デモに倣い **専用ディレクトリ + `index.html` + `<name>.js`**、`scripts/build-*.mjs` でビルド → `dist-*`
- TypeScript の `class` は使わない方針に従い、プレーンオブジェクト + 関数で構成

## 3. 機能要件（FR）

| ID | 要件 |
|----|------|
| FR-1 | **FPS視点**: クリックでポインターロック。マウス移動で yaw/pitch を操作。画面中央に照準（crosshair）。 |
| FR-2 | **プレイヤー移動**: WASD 移動 + Space ジャンプ（既存FPSコントローラを流用）。※下記「要確認」参照 |
| FR-3 | **飛行オブジェクト**: 複数個（既定8個程度、UIで増減可）が無重力でアリーナ内を漂い、壁で反射し続ける。 |
| FR-4 | **右クリック発射**: カメラ前方へ球（プロジェクタイル）を発射。一定速度で直進し、オブジェクト命中時に運動量（撃力）を与えて吹っ飛ばす。命中後/一定時間後に消滅。 |
| FR-5 | **左クリックでキャッチ**: 押した瞬間、照準方向のレイキャストで最も手前のオブジェクトを掴む。掴んだオブジェクトはカメラ前方の「手アンカー」とスプリングで接続される。 |
| FR-6 | **振り回し**: 左クリック保持中、視点移動に伴って手アンカーが動く。スプリング力でオブジェクトが遅れて追従し、振り回せる。 |
| FR-7 | **投擲（リリース）**: 左クリックを離すと、その瞬間のオブジェクト速度をそのままにスプリングを解除。勢いがあれば飛んでいき、壁で反射して再び漂う。 |
| FR-8 | **HUD/UI**: 照準・FPSカウンタ・操作説明オーバーレイ・「← 戻る」リンク。オブジェクト数などの簡易設定パネル。 |
| FR-9 | **アリーナ**: オブジェクトが逃げないよう閉じた箱型の空間（床・天井・四方の壁）。 |

## 4. 非機能要件（NFR）

- NFR-1: WebGPU 必須。非対応環境では既存デモ同様にエラーメッセージ表示（WebGLフォールバックは任意・後回し）。
- NFR-2: 60FPS を目標。物理ステップは固定タイムステップで安定動作させる。
- NFR-3: PC（マウス＋キーボード）専用。モバイル対応は今回スコープ外。
- NFR-4: 既存デモのコードスタイル・ファイル構成・UIデザインに合わせる。

## 5. スコープ外（今回やらないこと）

- スコア・勝敗・ゲームオーバー等のルール
- サウンド／エフェクト（パーティクル等）の作り込み
- モバイル／タッチ操作対応
- マルチプレイ、セーブ、ステージエディタ連携
- 物理エンジンの導入

## 6. 要確認事項（設計前に確定したい点）

1. **プレイヤー移動**: WASD で動き回れる方が良いか／その場固定で視点のみが良いか。
   - 既定案: **WASD移動あり**（既存コントローラ流用で実装コスト小・追いかけられて楽しい）
2. **オブジェクトの見た目**: 球のみ／箱・多面体など複数形状ミックス／カラフルに。
   - 既定案: **複数形状ミックス（球・箱・多面体）をカラフルに**
3. **重力**: 完全無重力で漂う／ゆるい重力で落ちる。
   - 既定案: **無重力で漂い、壁で反射**（「飛び回っている」イメージに合致）
4. **アリーナの見た目**: 既存FPSの階段ステージ流用／シンプルな箱型ルーム新規作成。
   - 既定案: **シンプルな箱型ルーム新規作成**（浮遊オブジェクトが映える）

> 上記「既定案」で問題なければそのまま設計（Stage 2）に進みます。変更したい項目があれば指定してください。
