# 要件定義書 - Cloth Preview

## 1. 目的

VRM モデルにマント（.cloth.json）を重ねた状態で VRMA アニメーションを再生し、
グリップイベント・ブレンドシェイプをタイムライン上で編集・確認できるスタンドアロンページを作成する。
作成したタイムラインデータは JSON で保存・読み込みでき、将来のエフェクト拡張にも対応できる設計とする。

---

## 2. スコープ

| 対象 | 内容 |
|------|------|
| 新規ファイル | `cloth-preview/index.html`, `cloth-preview/cloth-preview.js` |
| 流用 | cloth-editor.js の布シミュ・VRM 読み込み・ハンドグラブ実装 |
| 対象外 | 既存ページの変更、ゲームへの直接組み込み |

---

## 3. 機能要件

### FR-01: ファイル読み込み

| ID | 要件 |
|----|------|
| FR-01-1 | VRM ファイル（.vrm）をボタンまたはドラッグ&ドロップで読み込める |
| FR-01-2 | VRMA ファイル（.vrma）をボタンまたはドラッグ&ドロップで読み込める |
| FR-01-3 | マントファイル（.cloth.json / .json）をボタンまたはドラッグ&ドロップで読み込める |
| FR-01-4 | タイムラインデータ（.timeline.json）を読み込み・上書き保存できる |
| FR-01-5 | VRM 読み込み時にブレンドシェイプ（表情）一覧を自動取得しトラック候補として表示する |

### FR-02: ビューポート

| ID | 要件 |
|----|------|
| FR-02-1 | WebGPU renderer で VRM + マントを同一シーンに描画する |
| FR-02-2 | OrbitControls でカメラ操作できる |
| FR-02-3 | マントの布シミュレーションがリアルタイムで動作する（cloth-editor の実装を流用） |
| FR-02-4 | VRM の手ボーン位置にグラブポイント球（青=左、橙=右）を表示する |
| FR-02-5 | グラブポイント球をドラッグしてオフセット微調整できる |
| FR-02-6 | WebGPU 非対応時に警告を表示し布シミュを無効にする |

### FR-03: VRMA アニメーション再生

| ID | 要件 |
|----|------|
| FR-03-1 | VRMA を読み込むと AnimationMixer + createVRMAnimationClip で再生できる |
| FR-03-2 | 再生・停止・ループ・再生速度（0.25/0.5/1.0/2.0）を UI で制御できる |
| FR-03-3 | タイムラインのプレイヘッドドラッグでシーク（フレーム単位）できる |
| FR-03-4 | 再生中は VRM の手ボーンが動き、グラブポイントが自動追従する |
| FR-03-5 | アニメーション終端でループまたは停止する |

### FR-04: タイムライン表示

| ID | 要件 |
|----|------|
| FR-04-1 | 横軸 = フレーム（FPS 設定可能、デフォルト 30fps）、縦軸 = トラック一覧 |
| FR-04-2 | トラックは「グリップ」セクションと「ブレンドシェイプ」セクションに分かれる |
| FR-04-3 | プレイヘッドがリアルタイムに現在フレームを示す |
| FR-04-4 | フレームスケール（px/frame）をマウスホイールでズームできる |
| FR-04-5 | タイムライン全体を横スクロールできる |
| FR-04-6 | 総フレーム数（=アニメーション尺）が自動設定され、手動でも変更できる |

### FR-05: グリップイベントトラック

| ID | 要件 |
|----|------|
| FR-05-1 | トラック種別: `gripLeft`, `gripRight`, `releaseLeft`, `releaseRight`（各1行） |
| FR-05-2 | 各トラック行の任意フレームをクリックでイベントを配置・削除（トグル）できる |
| FR-05-3 | 再生中に該当フレームへ到達したら対応する clothAPI 関数を呼び出す |
| FR-05-4 | clothAPI が未初期化（布シミュ未開始）の場合はイベントをスキップする |

### FR-06: ブレンドシェイプトラック

| ID | 要件 |
|----|------|
| FR-06-1 | VRM から取得した表情名（preset + custom）を行として追加できる |
| FR-06-2 | 各行の任意フレームをクリックでキーフレームを配置し、値（0〜1）を設定できる |
| FR-06-3 | キーフレーム間は線形補間で値を計算し vrm.expressionManager.setValue() で適用する |
| FR-06-4 | VRMA のブレンドシェイプトラックと合算される（VRMA 優先） |
| FR-06-5 | キーフレームをクリックで選択し、値を数値入力で編集できる |
| FR-06-6 | キーフレームを右クリックまたは Delete キーで削除できる |

### FR-07: タイムラインデータの保存・読み込み

| ID | 要件 |
|----|------|
| FR-07-1 | タイムラインデータを `.timeline.json` として保存できる |
| FR-07-2 | 保存形式はイベントトラックとブレンドシェイプトラックを含む JSON |
| FR-07-3 | 読み込んだ `.timeline.json` を現在の状態に復元できる |
| FR-07-4 | フォーマットは将来の `effect` トラック追加を想定した拡張可能な構造にする |

---

## 4. 非機能要件

| ID | 要件 |
|----|------|
| NFR-01 | WebGPU renderer 使用（Three.js v0.184、esm.sh CDN） |
| NFR-02 | 外部依存は esm.sh CDN のみ（ビルドなし） |
| NFR-03 | TypeScript 不使用（cloth-editor と同方針の素 JS） |
| NFR-04 | VRM 非読み込み時は再生・タイムライン操作を無効にする |
| NFR-05 | ファイルはスタンドアロン 2 ファイル構成（index.html + cloth-preview.js）|

## 5. タイムライン JSON フォーマット（仕様）

```json
{
  "version": 1,
  "fps": 30,
  "durationFrames": 90,
  "tracks": [
    {
      "kind": "grip",
      "type": "gripLeft",
      "frames": [10, 45]
    },
    {
      "kind": "grip",
      "type": "releaseLeft",
      "frames": [60]
    },
    {
      "kind": "blendShape",
      "name": "happy",
      "keyframes": [
        { "frame": 0,  "value": 0.0 },
        { "frame": 15, "value": 1.0 },
        { "frame": 30, "value": 0.0 }
      ]
    },
    {
      "kind": "effect",
      "type": "someEffect",
      "frames": [20],
      "params": {}
    }
  ]
}
```

## 6. clothAPI インターフェース（流用）

cloth-editor.js で定義済みの `window.clothAPI` を同仕様で実装：

```js
clothAPI.gripLeft(x, y, z)    // 左手グリップ ON（手ボーン位置を自動渡し）
clothAPI.gripRight(x, y, z)   // 右手グリップ ON
clothAPI.releaseLeft()         // 左手グリップ OFF
clothAPI.releaseRight()        // 右手グリップ OFF
```

イベント発火時はグラブポイントの現在ワールド座標を自動的に引数として渡す。

---

## 7. 制約・前提

- VRM は `@pixiv/three-vrm@3.4.0`、VRMA は `@pixiv/three-vrm-animation` を esm.sh CDN で使用
- マントは cloth-editor で作成した `.cloth.json`（グリップ付き保存済み）を想定
- 布シミュは WebGPU 必須。非対応環境でも VRM + VRMA 再生は動作する
