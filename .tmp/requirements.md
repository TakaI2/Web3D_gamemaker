# 要件定義書 - ゲームフロー フェーズ2a：敵の攻撃とライフ

## 1. 目的

戦闘（swing-catch フロー戦闘モード）に「敵の攻撃手段」と「プレイヤーのライフ」を実装し、
lose ルートを実際に成立させる。敵は近接（挑発→大ダメージ）と遠距離（弾／浮遊オブジェクト投擲）で攻撃し、
被弾時は当てた NPC が専用セリフ（bark）をしゃべる。浮遊オブジェクトにも接触ダメージを付ける。

決定事項（ユーザー確認済み）:
- 近接＋遠距離の両方。
- 近接＝接近して挑発（予備動作）→ 踏み込み大ダメージ。
- 遠距離＝弾を撃つ／周囲の浮遊オブジェクトを掴んでこちらへ投げる。
- 浮遊オブジェクトは接触でダメージ（今は無反応）。
- 被弾時の専用セリフ＝当てた NPC がしゃべる（新イベント `attackHit`）。
- ライフ＝HPバー＋被ダメ演出。

スコープ: 戦闘モード（`?flow=1`）でのみ作動。**非フロー時の swing-catch サンドボックス挙動は不変**（後方互換）。
ボス（HP/フェーズ/専用攻撃）は **フェーズ2b** で別途。

## 2. 機能要件

### FR-01: 敵の戦闘 AI（戦闘モードで有効化）
| ID | 要件 |
|----|------|
| FR-01-1 | 戦闘開始時、各敵の behavior を「攻撃的」に上書き（検知/挑発を有効化）し、プレイヤーへ engage する |
| FR-01-2 | 各敵は近接/遠距離の攻撃を**クールダウン**付きで行う（同時多発しすぎないよう間隔・確率管理） |
| FR-01-3 | 敵がラグドール/被掴み/ダウン中は攻撃しない |
| FR-01-4 | 非フロー時はこの AI を作動させない（従来の受動挙動） |

### FR-02: 近接攻撃（挑発→大ダメージ）
| ID | 要件 |
|----|------|
| FR-02-1 | 敵が近接間合いへ接近し「挑発」（予備動作＝テレグラフ：寄る＋挑発 bark／モーション）を行う |
| FR-02-2 | 予備動作後に踏み込み、プレイヤー至近で**大ダメージ**を1回与える |
| FR-02-3 | 命中後はクールダウンに入り離脱する（連続大ダメージを防ぐ） |
| FR-02-4 | 予備動作中はプレイヤーが回避できる猶予がある（テレグラフ時間） |

### FR-03: 遠距離攻撃（弾）
| ID | 要件 |
|----|------|
| FR-03-1 | 敵がプレイヤー方向へ弾（敵プロジェクタイル）を発射する |
| FR-03-2 | 敵弾がプレイヤーに当たると小ダメージ。弾は消滅 |
| FR-03-3 | 敵弾はプレイヤー弾と区別して管理する（自分には当たらない） |

### FR-04: 遠距離攻撃（浮遊オブジェクト投擲）
| ID | 要件 |
|----|------|
| FR-04-1 | 敵が近くの浮遊オブジェクト（`objects`）を1つ掴み、プレイヤーへ射出する |
| FR-04-2 | 投擲されたオブジェクトがプレイヤーに当たると中ダメージ |
| FR-04-3 | 投擲後のオブジェクトは通常の浮遊物に戻る（破棄しない） |

### FR-05: 浮遊オブジェクトの接触ダメージ
| ID | 要件 |
|----|------|
| FR-05-1 | 戦闘中、浮遊オブジェクトがプレイヤーに接触したら小ダメージ＆跳ね返る（クールダウン付き） |
| FR-05-2 | 非フロー時は接触ダメージなし（従来通り無反応） |

### FR-06: 被弾セリフ（当てた NPC がしゃべる）
| ID | 要件 |
|----|------|
| FR-06-1 | 敵の攻撃（近接/弾/投擲オブジェクト）がプレイヤーに当たったら、当てた NPC が `attackHit` bark をしゃべる |
| FR-06-2 | `attackHit` を npc データの `events` に追加し、Character Editor のイベントセリフで編集できる |
| FR-06-3 | 投擲オブジェクト/弾の発生源 NPC を追跡し、命中時にその NPC が bark する |
| FR-06-4 | bark のクールダウンは既存仕様（1500ms）に従い連発しない |

### FR-07: プレイヤーライフ UI と被ダメ演出
| ID | 要件 |
|----|------|
| FR-07-1 | 画面に HP バー（最大＝`battle.lose.hp`）を表示（フェーズ1のHUDを拡張） |
| FR-07-2 | 被弾で HP 減少。攻撃種別ごとにダメージ量が異なる（近接=大/弾=小/投擲=中/接触=小） |
| FR-07-3 | 被弾時に赤フラッシュ（画面ふち）＋短いシェイク等の演出 |
| FR-07-4 | HP 0 で敗北（既存 endBattle('lose')） |

## 3. 非機能要件

| ID | 要件 |
|----|------|
| NFR-01 | 戦闘 AI/ダメージ/演出は `FLOW` ガードで隔離。非フロー時は一切作動しない（後方互換） |
| NFR-02 | ダメージ量・クールダウン・間合い等は定数で集約（ハードコード散在を避ける） |
| NFR-03 | 既存の lib（npc-speech/state-machine/ragdoll/cloth）を活かし重複実装しない |
| NFR-04 | 敵弾/投擲オブジェクトは多重生成で破綻しない（上限・寿命管理） |
| NFR-05 | TypeScript/class 不使用（既存方針） |

## 4. ダメージ/パラメータ（既定・後で調整可）

| 項目 | 既定値 |
|------|--------|
| 近接（大） | 3 |
| 敵弾（小） | 1 |
| 投擲オブジェクト（中） | 2 |
| 接触（小） | 1 |
| 近接テレグラフ時間 | 約1.0s |
| 近接間合い / 命中半径 | 接近〜2.5m / 1.5m |
| 攻撃クールダウン（敵ごと） | 2.5〜4s（種別で別） |
| 被ダメ無敵時間 | 0.8s |

---

## 5. 制約・前提
- 対象は swing-catch のフロー戦闘モード（`?flow=1`）。
- 既存の `attack`/`taunt` ステートと state-machine を活用（挑発＝taunt の延長、踏み込み＝attack）。
- `attackHit` イベントは既存 bark 機構（npc-speech）に種別追加するだけで動く（events に lines があれば発話）。
- ボス専用攻撃・HP/フェーズはフェーズ2b。
