プロシージャル配置の理論 — Room Editor と都市生成の仕組み

本資料は lib/room-gen.js（部屋＋家具）と、同じ設計思想の lib/kenney-buildings.js（都市の建物）が「どういう理論・式で物を配置しているか」をまとめたもの。

1. 全体像：決定的プロシージャル生成

生成は「シード → 乱数列 → ルール適用 → 配置リスト」の一方向パイプライン。同じ入力（type, w, d, seed, windowRate）なら必ず同じ部屋が出る（＝決定的）。これにより：

  • 保存が軽い：家具の全座標ではなくパラメータだけ保存しても再現できる
  • 「🎲でガチャ → 気に入ったシードを保存」というワークフローが成立する
  • ネットワーク同期や検証（node でのテスト）が容易
Room = G(type, w, d, seed, windowRate)　… G は純関数（副作用なし・three.js 非依存）

生成順は「動かせない物 → 大きい物 → 小さい物」：外殻（床・壁）→ 壁付け大型家具 → 中央 → コーナー → ラグ小物。これは後述の貪欲法で衝突を避けるための定石（大物ほど置ける場所が少ないので先に確保する）。

2. 乱数の理論：シード付きPRNG（mulberry32）

Math.random() はシードを与えられないため使わない。代わりに mulberry32（32bit整数ハッシュを反復する擬似乱数生成器）を使う：

function mulberry32(a) {
  return function () {
    a |= 0; a = (a + 0x6D2B79F5) | 0;              // 黄金比由来の増分（Weyl列）
    let t = Math.imul(a ^ (a >>> 15), 1 | a);       // xorshift + 乗算で撹拌
    t = (t + Math.imul(t ^ (t >>> 7), 61 | t)) ^ t;
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296;   // [0,1) へ正規化（÷2^32）
  };
}

状態は32bit整数1個。呼ぶたびに内部状態が進み、一様分布 [0,1) の値を返す。シードは事前にKnuthの乗法ハッシュで撹拌して、連番シード（1,2,3…）でも似た部屋にならないようにする：

rng = mulberry32( (seed × 2654435761) mod 2³² )　… 2654435761 ≈ 2³² / φ（黄金比）

派生する基本ツール：

一様整数：irange(a,b) = a + ⌊rng() × (b−a+1)⌋
一様選択：pick(arr) = arr[⌊rng() × len⌋]
確率イベント：rng() < p なら発生（ベルヌーイ試行）

3. 座標系：セルグリッドと実寸変換

部屋は w × d のセル格子で表す（1セル＝床タイル1枚）。ジェネレータはセル座標のみを扱い、実寸（メートル）を知らない。実寸換算はエディタ側が床タイルの実物を測って行う：

TILE = max(bboxₓ, bbox_z) of floorFull.glb　（実測）
ワールド座標 = セル座標 × TILE

この分離により、キットの縮尺が変わってもジェネレータは無修正で済む（解像度非依存）。同じ発想は都市生成（1セル=道路タイル1枚）や、plateau-fly の建物正規化（フットプリント実寸→スケール逆算）でも使っている。

4. 外殻の生成：外周走査・ドア・窓

床は全セルに floorFull を敷く（二重ループ）。壁は外周セルを4辺に分けて走査し、辺ごとの向きベクトル表 SIDE に従って「セル中心から外側へ半セル」ずらした位置に置く：

壁位置 = (x + dx·0.5, z + dz·0.5)、 (dx,dz) は辺の外向き法線
北: (0,−1) ry=0　東: (1,0) ry=−π/2　南: (0,1) ry=π　西: (−1,0) ry=π/2
  • ドア：南面（手前）の辺セルから一様に1つ選び wallDoorway(Wide) に差し替える。ドアセルとその内側1セルを占有グリッドで「予約」し、家具で塞がないようにする（クリアランス）。
  • 窓：残りの各壁スロットで rng() < windowRate のベルヌーイ試行。成功なら窓壁に差し替え。期待窓数 ＝ 壁数 × windowRate。
  • 四隅：wallCorner を (−0.5,−0.5) など角の外側点に、90°ずつ回して置く。

5. 占有グリッドと衝突判定（矩形AABB）

重なり防止は w×d の Uint8Array（0=空き / 1=占有 / 2=ドア前予約）で管理する。家具はフットプリント fp=[fw,fd]（セル数）の矩形として扱い、置けるかは全被覆セルの空き判定：

canPlace(x, z, fw, fd) =
  0 ≤ x, z  かつ  x+fw ≤ w, z+fd ≤ d          … 部屋内
  かつ ∀(i,j)∈[0..fw)×[0..fd): occ[z+j][x+i] == 0   … 全セル空き

置いたら mark() で全セルを1にする。これは離散化した AABB（軸平行境界箱）判定で、O(fw×fd) と高速。連続座標の当たり判定より粗いが、「家具は概ねセル単位」という前提では十分で、確実に非重複を保証できる。

ラグ・ドアマットは occupy:false でグリッドに書き込まない＝重なり許可レイヤー（上にテーブルを置ける）。

6. 家具配置：ルール表と貪欲法（greedy packing）

部屋タイプごとの配置は宣言的なルール表（データ）で持つ。1ルール＝「候補モデル群・置き方・個数範囲・フットプリント」：

{ models:['loungeSofa','loungeSofaLong'], place:'wall', count:[1,1], fp:[2,1] }
{ models:['tableCoffee', …],              place:'center', count:[1,1] }
{ models:['pottedPlant', …],              place:'corner', count:[0,2] }

コード（アルゴリズム）とデータ（部屋らしさ）が分離されているので、部屋タイプの追加・調整はルール表の編集だけで済む。配置は貪欲法：ルールを上から順に処理し、それぞれ候補位置を試して最初に置けた場所で確定（後戻りしない）。

place の種類と候補列挙

place候補の作り方向き ry
wall外周に接する内側セルを全列挙 → Fisher–Yates でシャッフル → 順に canPlace壁の内向き法線（部屋の中心を向く）
corner四隅 (0,0)(w−1,0)(0,d−1)(w−1,d−1) をシャッフルランダム
center((w−fw)/2, (d−fd)/2) を最初に試行0
free一様乱数の位置を最大14回試す（棄却サンプリング）0

壁付けでは、フットプリントを壁に沿って回転させる必要がある。北南の壁なら fp=[fw,fd] のまま、東西なら入れ替え：

壁沿い footprint = alongX ? [fw, fd] : [fd, fw]　（90°回転で幅と奥行が入れ替わる）

シャッフルは Fisher–Yates（一様なランダム順列。i を後ろから回し j∈[0..i] と交換）。これにより「壁のどこに置かれるか」がシードごとに変わり、かつ全位置が等確率で試される。

なぜ貪欲法で十分か：部屋の家具数（5〜12個）に対しセル数（20〜200）が十分多く、最適詰め込み（NP困難なパッキング問題）を解く必要がない。「置けなければそのルールを打ち切る」ことで、狭い部屋では自然に家具が減る＝破綻しない。

7. 向きと付属配置の式

前方1セル（デスクの椅子・テーブルの椅子）

親の向き ry から前方ベクトルを求め、1セル先に子を置き、子は親を向く（ry+π）：

front = (sin ry, cos ry)　… y軸回転 ry の +Z 方向
子の位置 = 親 + front、子の向き = ry + π

台の上に載せる（stackOn）

TV・PC画面など。「載せる」はy座標＝台の高さで表現し、高さはランタイムが台モデルの bbox から実測：

y_子 = bboxHeight(stackOn モデル)　（例: televisionModern の y = cabinetTelevision の高さ）

テーブルの椅子（周回配置）

テーブル矩形の4辺中点の外側1セルを候補にし、シャッフル後に各候補を確率0.7で採用 → 椅子はテーブル中心を向く。個数・配置がシードごとに揺らぎ「生活感」が出る。

キッチンのカウンター列（系列生成）

単品配置ではなく決まった系列（冷蔵庫→カウンター→シンク→引き出し→コンロ→…）を北壁沿いに左から詰める。現実のキッチンは「並び」に意味があるため、乱数ではなく文法（シーケンス）で表現する方が自然になる。

8. 応用：都市の建物配置（kenney-buildings.js）

同じ理論を街スケールに適用したのが plateau-fly の建物配置。違いは「壁沿い」が「道路沿い」になる点：

道路に沿った等間隔サンプリング

道路エッジ (A→B) を長さ L とすると、配置点数 n = ⌊L / spacing⌋
t = i/n (i=1..n−1) で線分を内分 → P(t) = A + (B−A)·t　（端点＝交差点は避ける）

道路からの後退（setback）と建物の向き

垂直ベクトル perp = (−dz, dx)　… 道路方向 (dx,dz) を90°回転
建物中心 = P(t) ± perp × setback
setback = 道路半幅 + フットプリント/2 × maxScale + 余白　… 車道を絶対に塞がない
建物の向き ry = atan2(−perp·side) … 正面が道路を向く

占有グリッド（部屋と同じ）

6mセルの疎な占有集合（Set<"gx_gz">）にフットプリント半径ぶんをマークし、建物同士の重なりを防ぐ。部屋の Uint8Array と同じ理論の、無限平面向け実装（ハッシュセット化）。

ゾーニング（中心距離による分類）

dist = √(x² + z²)　→　dist < 500m: 高層 ／ < 1100m: 中層 ／ それ以遠: 住宅

さらにセル座標のハッシュでシードを作る hash3(seed, gx, gz) により、建物1軒ごとに独立した決定的乱数を持たせている（どの順で生成しても同じ結果になる＝順序非依存）。

9. 用語集

用語意味
決定的生成同じシードから常に同じ結果が出ること。保存・再現・テストの基盤
PRNG擬似乱数生成器。mulberry32 は32bit状態の軽量版
ベルヌーイ試行確率 p で成功するコイン投げ。窓の有無・椅子の間引きに使用
棄却サンプリングランダム候補を出し、条件を満たすまで捨てて引き直す（上限つき）
Fisher–Yates一様なランダム順列を O(n) で作るシャッフル
貪欲法局所的に最初に成立した解を採用し後戻りしない。高速で破綻しにくい
AABB軸平行の矩形（箱）。占有判定を単純な範囲比較にできる
フットプリント物が占める床面のセル矩形 [幅, 奥行]
クリアランスドア前など「機能のために空けておく」予約領域
ゾーニング位置（中心距離など）で配置ルールを切り替える区画化
